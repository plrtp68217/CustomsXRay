# AGENTS.md

## Контекст решений проекта CustomsXRay

---

В этом файле фиксируются все принятые решения: архитектурные, технологические, дизайнерские — с обоснованием, почему выбран именно этот вариант.

---

### Принятые решения

#### [2026-06-22] Структура ведения проекта
- **Решение:** Созданы два файла — `LOG.md` (журнал инструкций и описаний) и `AGENTS.md` (контекст решений).
- **Причина:** Разделение операционного журнала (что делать) и памяти решений (почему так) помогает сохранять контекст между сессиями и не терять обоснования выбранных подходов.
- **Альтернативы:** Единый файл журнала — отвергнут, так как смешение задач и обоснований затрудняет поиск нужной информации.

#### [2026-06-22] Рендеринг на HTML5 Canvas вместо DOM/Drag библиотек
- **Решение:** Игровые сцены (укладка силуэтов, рентген-лента) рендерить на `<canvas>` с ручной обработкой pointer-событий, а не через HTML Drag-and-Drop API.
- **Причина:** Нужна точная геометрия (полигоны, поворот, расчёт площади перекрытия силуэтов), плавная анимация и единый конвейер отрисовки. DOM+DnD плохо справляется с вращением и sub-pixel совмещением.
- **Альтернативы:** SVG + Drag — приемлемо, но расчёт перекрытия сложнее; библиотеки типа fabric.js — лишняя зависимость, противоречит требованию лёгкости.

#### [2026-06-22] Авторитарный сервер + shared-модули геометрии/предметов
- **Решение:** `geometry.js` и `items.js` выполнены shared (UMD-стиль: `module.exports` для Node, `window.*` для браузера). Сервер пересчитывает % совпадения по присланным координатам самостоятельно.
- **Причина:** Защита от читерства — клиент не может заявить высокий % совпадения; сервер считает сам по тем же формулам. Единый источник правды для формы предметов.
- **Альтернативы:** Дублировать данные на сервере и клиенте — рассинхрон; доверять клиенту — нарушает требование валидации.

#### [2026-06-22] Модель «полость = форма контрабанды»
- **Решение:** Полость внутри легального предмета имеет ту же форму, что и контабандный силуэт. Игрок совмещает контрабанду с полостью; % совпадения = площадь перекрытия / площадь контрабанды.
- **Причина:** Даёт чёткую измеримую метрику 0–100% и narrativно обосновано: идеально вставленная контрабанда заполняет пустоту и на рентгене не видна, плохо вставленная — выпирает в «плотные» зоны предмета и светится красным.
- **Альтернативы:** Произвольная форма полости — сложнее математически и менее интуитивно для игрока.

#### [2026-06-22] Видимость контрабанды на рентгене = (1 − match)
- **Решение:** Во фазе проверки контрабанда рисуется красным с opacity = (1 − match) × k; при match≥85% почти невидима, при низком — яркая + CSS-вибрация контейнера.
- **Причина:** Связывает мастерство контрабандиста с заметностью для таможенника —核心 напряжения игры.
- **Альтернативы:** Пороговая (видно/не видно) — менее интересно, убирает градиент навыка.

#### [2026-06-22] Таймеры на сервере
- **Решение:** Сервер запускает фазы по `setTimeout`, присылает `endTimeMs`; клиент рисует обратный отсчёт локально, но фаза заканчивается по серверному событию.
- **Причина:** Защита от рассинхрона/подделки таймера клиентом; единое завершение фазы для обоих игроков.
- **Альтернативы:** Клиентские таймеры — рассинхрон и риск читерства.

#### [2026-06-22] Подсчёт очков
- **Решение:** За раунд: таможенник +1 за изъятие именно контрабанды; контрабандист +1, если контрабанду не изъяли; таможенник −1 за каждое ложное изъятие чистого предмета (можно изымать несколько раз до попадания/таймаута).
- **Причина:** Соответствует промпту и поощряет аккуратность таможенника.
- **Альтернативы:** Единичная попытка изъятия — слишком жёстко, мало геймплея.

#### [2026-06-22] Тестирование: smoke + Playwright e2e
- **Решение:** Два уровня тестов: `test_flow.js` — Node smoke-тест на socket.io-client (2 подключения, полный цикл, проверка счётчиков фаз); `e2e/full-flow.spec.js` — Playwright с двумя реальными браузерами (хост+гость), прогон 3 раундов в двух сценариях (таможенник изымает / бездействует). Для быстрых e2e — отдельный `e2e-server.js` с короткими таймерами; таймеры в `server.js` вынесены в env-переменные.
- **Причина:** Smoke быстро ловит регрессии серверной логики без браузера; Playwright проверяет реальный UI/Canvas-интерактив и весь SPA-поток экранов. Разделение «быстрый серверный» + «полный браузерный» даёт coverage без медленных full-stack тестов на каждый чих.
- **Альтернативы:** Только Playwright — медленно для частого прогона; только smoke — не покрывает клиентский рендер и UX-переходы.

#### [2026-06-22] Канвас на absolute внутри stage (фикс роста высоты)
- **Решение:** `canvas { position:absolute; inset:0; width:100%; height:100% }` внутри `.canvas-stage` (position:relative); `fitCanvas` читает `parent.clientWidth/clientHeight` и не задаёт inline-стиль высоты.
- **Причина:** Прежний код ставил `canvas.style.height = parent.getBoundingClientRect().height` (border-box), а высота stage была контентно-зависимой от канвasa → петля +2px (бордюр) на каждый fitCanvas → экран «рос» при Drag и в rAF-цикле рентгена.
- **Альтернативы:** Фиксированная высота stage — негибко на разных экранах.

#### [2026-06-22] Раскладка рентгена без перекрытий + hit-test выступающей контрабанды
- **Решение:** `computeScanLayout` раскладывает предметы по их реальным ширинам с gap, при нехватке ширины равномерно масштабирует; предметы статичны (движутся только полосы ленты). `itemAtPoint` помимо легального полигона hit-testит полигон контрабанды для контрабандного предмета.
- **Причина:** Баг 3 (перекрытия при крупных предметах/узком канвасе) и баг 2 (контрабанда вне легального силуэта была не кликабельна).
- **Альтернативы:** Равномерные слоты `w/(n+1)` — порождали перекрытия; масштабирование только рентген-сетки — не решало кликабельность выступающей части.

#### [2026-06-22] Ручка поворота на самом силуэте (предложение)
- **Решение:** На контрабандном силуэте рисуется ручка `↻` (кружок сверху по центру bbox). Pointerdown на ручке → режим вращения; горизонтальный Drag меняет `placement.rot` (вправо — по часовой, влево — против). Нижняя кнопка и стрелки сохранены как фолбэк.
- **Причина:** Запрос пользователя: вращение должно быть «на самом предмете» с drag влево/вправо.
- **Альтернативы:** Жест двумя пальцами — ненадёжно на десктопе; ползунок угла — лишний UI.
#### [2026-06-22] Mute через master-gain + остановка активных звуков
- **Решение:** `Audio.setEnabled(v)` выставляет `master.gain.setTargetAtTime(v?0.5:0)` и при mute вызывает `stopScannerHum`; `init` ставит `master.gain.value = enabled?0.5:0`. В onclick кнопки при unmute во время рентгена гул перезапускается.
- **Причина:** Прежний `setEnabled` только блокировал новые звуки — уже играющий гул сканера не замолкал, плюс `init` после mute сбрасывал gain в 0.5.
- **Альтернативы:** Проверять `enabled` в каждом osc — не глушит уже запущенные; отдельный mute-узел — избыточно.

#### [2026-06-22] Конвейерная петля для рентгена (предметы едут, без перекрытий)
- **Решение:** `computeScanLayout` → `{L=w+maxWS, slot=L/n, scale, maxWS}`; масштаб подбирается из `(w - n*gap)/(n-1) ≥ maxW*scale`. Позиция `cx = (i*slot + conveyorX) mod L`, скорость `conveyorX += 3.0/кадр`. Предметы качаются (rot ±0.07, bob ±5px).
- **Причина:** Баг «предметы лежат, лента едет» + требование «машина едет с товарами и качается». Петля с mod L даёт бесшовное движение и (за счёт slot ≥ maxWS) гарантированно без перекрытий на любой ширине.
- **Альтернативы:** Равномерные слоты `w/(n+1)` — порождали перекрытия; tile-повтор группы — дублировал бы предметы.

#### [2026-06-22] Болтанка контрабанды = «шансы для всех» (99% тоже найти можно)
- **Решение:** Каждый кадр `curMatch = computeMatch(contraband+wobble, cavity)` в локальных координатах (вращение корпуса взаимно сокращается). `opacity = (1-curMatch)*0.95`. Wobble: wdx ±9px, wdy ±6px (синусы по времени + фаза предмета). Качание корпуса (rot, bob) — визуальное, на match не влияет.
- **Причина:** При 99% статического match контрабанда была неотличима → таможенник не имел шанса. Болтанка («груз качается в кузове») периодически выводит контрабанду из полости → curMatch падает → красный силуэт появляется. Так шанс есть у таможенника всегда, но у хорошего контрабандиста protrusion короче/слабее.
- **Альтернативы:** Пороговая видимость с шумом — менее физично; зависимость opacity от случайного штрафа — непредсказуемо.
